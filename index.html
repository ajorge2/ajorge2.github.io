02 CodeArchitect
Node · Express · React · Vite · tree-sitter · Anthropic API

An interactive tool for reading a codebase by its structure. Every file is parsed into a tree and drawn as nested, selectable pieces — function, block, expression — over the source, with a single slider that zooms continuously from whole-file down to individual tokens, re-chunking live as you drag.

Each piece carries a stable identity derived from the tree rather than the current zoom, so understanding accumulates across re-chunking instead of being discarded. Meanings are computed recursively and cached against those identities — a snippet's standalone meaning is computed once and reused wherever it recurs, and different slicings of a file share their common ancestors — then folded with surrounding scope up through the folder tree, so an answer about a single line can draw on the whole project.

Question-answering is retrieval the user steers: the selected piece is the unit, so there is no vector database — the access pattern is exact-key lookup along one path, not similarity search. The meaning cache is content-addressed and therefore self-invalidating (edited code yields new keys; stale entries are simply never read again), backed by a plain on-disk file with an in-memory layer rather than any standing service.

03 OrcView
Python · Flask · SSE · SQLite · Anthropic API · vanilla JS

An observability layer for LLM agents. Real-time ReAct visualization, hierarchical drill-down into nested sub-agent runs, and SSE streaming of every state transition, tool call, and model response.

Cycle detection threads an immutable frozenset call chain through every sub-agent invocation, preventing recursive loops without locks and without blocking parallel sub-agent execution. SQLite persists versioned run history with config snapshots, so agent behavior can be compared across model, prompt, and tool changes.

04 Learning Accelerator
FastAPI · SQLite · OpenAI structured outputs · Vite · D3.js · Algolia

A personal knowledge-graph app for mapping a subject as a network of ideas and relationships. The model generates the initial graph from notes or an uploaded PDF, then expands and corrects it via a typed GraphDelta diff schema (add, update, delete) rather than regenerating the full graph.

A reflection mode walks the graph breadth-first, prompting the user to articulate each concept; those articulations feed back into the next graph update, closing the loop between user understanding and graph structure. Every model call is typed via Pydantic schemas, so malformed responses fail loudly instead of silently corrupting state.
